Licensing System - Installation & Configuration Guide

This guide walks you through installing, configuring, and securing the Licensing System. Follow these steps to get your system up and running smoothly.

1. Prerequisites

Before you begin, ensure that your environment meets the following requirements:

  • PHP 7.4+ installed
  • MySQL or MariaDB database
  • Composer (for PHP dependency management)
  • SMTP server details for email notifications
  • PayPal or Stripe credentials for payment gateway (optional)

2. Installation Steps

Follow these steps to install the Licensing System on your server or local machine:

  1. Clone the repository: 
    git clone https://github.com/your-repo/licensing-system.git
  2. Navigate to the project folder: 
    cd licensing-system
  3. Install Composer dependencies: 
    composer install
  4. Copy the .env.example file to .env: 
    cp .env.example .env
    This file will store your environment variables such as database credentials and API keys.
  5. Set up your database by importing the schema: 
    mysql -u root -p licensing_system < schema.sql
    Or use phpMyAdmin or another MySQL client to import the `schema.sql` file.

3. Configuration Steps

After installation, follow these steps to configure the system:

3.1 Configure the .env File

The .env file contains sensitive configurations, including database credentials, payment settings, and SMTP details. Open the .env file in a text editor and configure the following settings:

  • Database Settings: Set the database host, name, username, and password.
    • 
DB_HOST=localhost
DB_NAME=licensing_system
DB_USERNAME=root
DB_PASSWORD=yourpassword
  • SMTP Settings: Configure your SMTP server for sending emails (order confirmations, license activations, etc.).
    • 
SMTP_HOST=smtp.yourmailserver.com
SMTP_USERNAME=your-email@example.com
SMTP_PASSWORD=your-email-password
SMTP_PORT=587
SMTP_SECURE=tls
  • Payment Gateway Settings: Add your PayPal or Stripe credentials for payment processing.
    • 
PAYPAL_CLIENT_ID=your-paypal-client-id
PAYPAL_SECRET=your-paypal-secret
  • JWT Secret Key: Set a secure secret key for JWT authentication.
    • 
JWT_SECRET_KEY=your-secret-key

3.2 Configure the Database Connection

If you're using MySQL or MariaDB, ensure that your database credentials in the .env file match the actual database you created during installation. Ensure the database server is running and accessible from your project folder.

3.3 Set up PayPal or Stripe Payment Gateway (Optional)

If you're using PayPal or Stripe for payments, ensure that the credentials in the .env file are accurate:

  • PayPal: Set your PayPal client ID and secret in the `.env` file.
  • Stripe: Similarly, set your Stripe keys for payment processing.

3.4 Configure Email Notifications

For email notifications (e.g., order confirmations, license activations), ensure that your SMTP settings are correctly configured in the .env file. You can use services like Gmail, SendGrid, or your own SMTP server.

3.5 Configure SSL for Secure Connections

Ensure that your site is served over HTTPS by enabling SSL. This is critical for securing sensitive data, especially during user logins and payment transactions. If you're using Apache, you can enable SSL using the following command:

sudo a2enmod ssl && sudo systemctl restart apache2

4. Test the System

After completing the configuration steps, access the system in your browser:

  1. Open your browser and go to http://localhost (or the domain where you hosted the system).
  2. Log in using the admin credentials you created during installation, and navigate through the admin panel to manage products, orders, and clients.
  3. Test the client panel by logging in as a client and verifying that the order, license management, and ticket systems work properly.

5. Troubleshooting

If you encounter any issues during installation or configuration, check the following:

  • Database Connection Errors: Ensure that your database credentials in the .env file are correct and that the database server is running.
  • Payment Gateway Errors: Double-check your PayPal or Stripe credentials in the .env file.
  • SMTP Errors: Ensure that your SMTP credentials are correct and that your email server allows third-party applications to send emails.

6. Next Steps

Once the system is running and fully configured, you can:

  • Integrate additional payment methods (e.g., Bitcoin, bank transfer).
  • Enhance security features like two-factor authentication (2FA) for clients and admins.
  • Scale the system for production by setting up a proper web server (e.g., Apache or Nginx) and a cloud database.
